## Core types (current)
- **`trade_flows`** (`etl/tradepulse_etl/db.py`) — IMPLEMENTED (SQLite). The Layer-1 fact table.
  PK(reporter,partner,hs6,period,flow); value_usd + quantity + source + published_date.
  Populated by `pipeline.run` from the source seam (fixture now, Comtrade later).
- **`signals`** (`etl/tradepulse_etl/db.py`) — IMPLEMENTED. Filled by `signals.compute_signals`
  (PURE, deterministic). One row per (reporter,hs6,flow,period) clearing the §6 floors; band ∈
  {minor,moderate,significant,surge,collapse,new}.
- **`companies`** (Layer 2) — IMPLEMENTED as curated JSON content: `content/companies/pellets.json`
  (id, name, role, country, hs6[], profile_url, evidence_source, evidence_url, verified_date;
  **no contact fields** — Golden Rule). Read by `web/app/lib/companies.js`. Moves to a table later.
- **`requirement_pages` / `requirement_items`** (Layer 3) — IMPLEMENTED as curated JSON:
  `content/requirements/pellets-{jp,kr,eu}.json` (§8 template + change_log). `web/app/lib/requirements.js`
  DROPS any item missing source_url+verified_date. Git history = the change-log/audit trail.
- **`watches` / `alerts` / `locked_page_clicks`** (Layer alert engine) — IMPLEMENTED as append-only
  NDJSON capture (`data/watches.ndjson`, `data/alerts.ndjson`, `data/locked_clicks.ndjson`, gitignored).
  Alert logic is PURE in `etl/tradepulse_etl/alerts.py`. Move to tables + real email/Zalo delivery later.
- Remaining tables below are still PLANNED.

### Web snapshot contract (`web/public/data/snapshot.json`, generated by `export.py`)
**Country-centric (ADR-0003).** The Next.js app reads this file (seam). One product; both flows. Shape:
`{ generated_at, hs6, product{name_en,name_vi}, latest_period, is_sample, sources[],
countries[ {code, name_en, name_vi,
  exp{value_usd,period,yoy_delta,band,direction,history[{period,value_usd}]} | null,
  imp{ …same… } | null } ],
feed[ {code,name_en,name_vi, flow:'export'|'import', value_usd,yoy_delta,band,direction,period} ] }`.
`exp` = flow X, `imp` = flow M. Consumed by `web/app/lib/snapshot.js` → SSR by `web/app/page.js`
(map + toggle + feed) and `web/app/country/[code]/page.js` (drill-down). Country names from the
bundled Comtrade reference (`etl/tradepulse_etl/reference/countries.json`).
**One file per product:** `snapshot-<hs>.json` for each `config.COVERED_HS`; `snapshot.json` = the
landing default (pellets). Web `loadSnapshot(hs)` picks the file; the category switch is a file swap.

## Signal computation (deterministic — plan §6)
- Granularity: reporter country × HS-6 × quarter, split by flow.
- Comparison: **YoY only** (same quarter last year) — never QoQ (seasonal products).
- `delta = (value_this − value_same_period_last_year) / value_same_period_last_year`.
- Noise floors (all must pass): value ≥ $10M this qtr; base ≥ $2M; ≥ 4 quarters history.
- Bands: ±15–30 moderate · ±30–60 significant · beyond ±60 surge/collapse · base≈0 & now ≥$5M = new lane.
- Suppress −15…+15% ("minor" excluded by design). Declines are signals too.
