### follow-up (same day) — borders as ONE GL line buffer (real lag fix)
Zoom-gating alone wasn't enough: react-globe.gl `pathsData` builds a separate fat-line object PER RING
— 50m = 1629 rings = 1629 draw calls → still laggy when shown.
- Replaced `pathsData` with a single `THREE.LineSegments` (one BufferGeometry of ~98k segments = ONE
  draw call). Vertices placed via `getCoords(lat,lng,alt)` so they align with the texture + signal points.
- Built lazily on first zoom-in, added to `scene()` once, toggled by `.visible` (no React re-render,
  no rebuild). Disposed on unmount. Named three imports (BufferGeometry/Float32BufferAttribute/
  LineBasicMaterial/LineSegments) — not `import *`. Console clean; compile ~10s.

### follow-up (same day) — borders only at country zoom (fix lag)
Always-on 50m borders made the globe laggy (241 filled polygons rendered every frame, even zoomed out).
- **Zoom-gated:** `showBorders` state toggled from camera distance with hysteresis (show < 190, hide
  > 250). `pathsData = showBorders ? borderPaths : []` — the default/zoomed-out globe renders ZERO
  border geometry (smooth, original look); borders appear only at country-level zoom and vanish on
  zoom-out. Toggle runs on interaction-end + wheel, not per-frame.
- **Lighter geometry:** country outlines as LINE paths (`pathsData`, rings → polylines) instead of
  filled `polygonsData` — much cheaper to render when shown.

### follow-up (same day) — globe: one consistent texture + vector borders
Owner: on zoom the globe changed appearance (ocean/land colour) and showed no country boundaries.
Root cause: the lazy hi-res swap loaded a DIFFERENT-source texture (Solar System Scope daymap) whose
palette differs from the NASA blue-marble base → visible colour/water shift; and blue-marble imagery
has no borders, so zooming revealed no lines.
- **One texture at every zoom:** dropped the texture swap entirely (removed `earth-8k.jpg`). The globe
  is always the same blue-marble → never changes appearance; zoom-out = the original look. (Checked
  matching NASA blue-marble hi-res options: the Dec-2004 5400px matches ocean/land tone but adds winter
  snow the summer base lacks → still a visible shift, so not used. A season-matched 8k blue-marble would
  be the only way to add texture pixels without changing appearance; not sourced here.)
- **Clear boundaries:** added crisp **vector country borders** from world-atlas 50m as globe polygons
  (transparent cap/side, light stroke) — sharp at any zoom, cheap, geometry already in the bundle.
- **Sharpness:** kept max anisotropic filtering on the base texture (best a single texture can give at
  grazing angles). Console clean; no runtime errors.
